Chart

The Chart component plots data originating from it's attached Data components. Five different Chart types are available, including Line, Area, Scatter, Bar and Pie. The Chart component itself has various properties that change the appearance of the Chart, such as Labels, LegendEnabled, Description and GridEnabled.

Elements
List of Choices to Display (as a list)
ElementsFromString
Comma separated list of choices to use

Properties

BackgroundColor
Sets or gets the background color of the chart.
Description
Sets or gets the title of the chart.
GridEnabled
Hides or shows the grid of the chart. Only applies to Charts which have axes.
Height
Sets or gets the height of the chart, in pixels.
Labels
Indicates the custom Labels to apply to the x axis of the Chart. Values are taken in order (first label corresponds to the 0th x value, the second to the 1st, and so on) If a label is not provided, the default x value is used. Only takes effect on Charts which have axes.
LabelsFromString
Comma separated list of Labels to use for the x axis of the Chart. Only takes effect on Charts which have axes.
LegendEnabled
Hides or shows the legend of the chart.
PieRadius
Sets the Pie Radius of a Pie Chart from 0% to 100%, where the percentage indicates the percentage of the hole fill. 100% means that a full Pie Chart is drawn, while values closer to 0% correspond to hollow Pie Charts.
Type
Sets the type of the Chart to use. Determines the type of data to be plotted. Valid types include line, area, scatter, bar and pie.
Visible
Sets or gets whether the component should be visible on the screen. Value is true if the component is showing and false if hidden.
Width
Sets or gets the width of the chart, in pixels.

ChartData2D

A ChartData2D component represents a single two dimensional Data Series in the Chart component, for example a single Line in the case of a Line Chart, or a single Bar in the case of a Bar Chart. The Data component is responsible for handling all of the data of the Chart. The entries of the Data component correspond of an x and a y value.

The component is attached directly to a Chart component by dragging it onto the Chart.

Properties

Color
Sets or gets the color of the data series.
Colors
Sets or gets the colors of the data series. The colors are applied to the values in order, and are alternated in order if there are more entries than colors.
DataFileXColumn
Value used when importing data from a DataFile component Source. The value represents the column to use from the DataFile for the x entries of the Data Series. For instance, if a column's first value is "X", and a column value of "X" is specified, that column will be used for the x values.
DataFileYColumn
Value used when importing data from a DataFile component Source. The value represents the column to use from the DataFile for the x entries of the Data Series. For instance, if a column's first value is "Y", and a column value of "Y" is specified, that column will be used for the x values.
DataSourceValue
Key value to use for retrieving data from the selected Chart Data Source.
For TinyDB and CloudDB, this is the tag value.
For the AccelerometerSensor, the value should be one of the following: X Y or Z
For the GyroscopeSensor, the value should be one of the following: X Y or Z
For the LocationSensor, the value should be one of the following: latitude, longitude, altitude or speed
For the OrientationSensor, the value should be one of the following: pitch, azimuth or roll
For the Pedometer, the value should be one of the following: WalkSteps, SimpleSteps or Distance
For the ProximitySensor, no value is required.
For the BluetoothClient, the value represents the prefix to remove from the value. For instance, if values come in the format "t:12", the prefix can be specified as "t:", and the prefix will then be removed from the data. No value can be specified if purely numerical values are returned.
ElementsFromPairs
Comma separated list of Elements to use for the data series. Values are formatted as follows: x1,y1,x2,y2,x3,y3. Values are taken in pairs, and an entry is formed from the x and y values.
Label
Sets or gets the label of the data series.
LineType
Sets the type of lines to use for Line data series. Valid types include linear, curved or stepped.
PointShape
Sets the shape of points to use for the Scatter data series. Valid types include circle, square, triangle, cross, x.
Source
Sets the Source to use for the Data component. Valid choices include TinyDB, CloudDB, DataFile, Web, AccelerometerSensor, BluetoothClient, GyroscopeSensor, LocationSensor, OrientationSensor, Pedometer and ProximitySensor. The Source value also requires valid DataSourceValue, WebColumn or DataFileColumn properties, depending on the type of the Source attached (the required properties show up in the Properties menu after the Source is changed).
WebXColumn
Value used when importing data from a Web component Source. The value represents the column to use from the Web for the x entries of the Data Series. For instance, if the contents of the Web are retrieved in JSON format, and an array with the "X" tag exists, the "X" column value can be specified to use that array.
WebYColumn
Value used when importing data from a Web component Source. The value represents the column to use from the Web for the y entries of the Data Series. For instance, if the contents of the Web are retrieved in JSON format, and an array with the "Y" tag exists, the "Y" column value can be specified to use that array.

Methods

AddEntry(text x, text y)
Call this method to add an entry with the specified x and y value. Values can be specified as text, or as numbers. For Line, Scatter, Area and Bar Charts, both values should represent number. For Bar charts, the values are rounded to the nearest integer. For Pie Charts, the x value is a text value.
ChangeDataSource(component source, text keyValue)
Changes the Data Source of the component to the specified component Source with the specified key value. See the Source property for applicable components. See the DataSourceValue property for the interpretation of the keyValue. In the case of the DataFile and Web components, the keyValue is expected to be a CSV formatted string, where the first value corresponds to the x column, and the second value corresponds to the y value.
Clear
Call this method to remove all the entries from the data series.
boolean DoesEntryExist(text x, text y)
Returns a boolean value specifying whether an entry with the specified x and y values exists. The boolean value of true is returned if the value exists, and a false value otherwise. See AddEntry for an explanation of the valid entry values.
list GetAllEntries
Returns all entries of the data series. The returned value is a list, where each element of the list is a list containing 2 elements -- one for the x value, and one for the y value.
list GetEntriesWithXValue(text x)
Retruns all entries of the data series matching the specified x value. The returned value is a list, where each element of the list is a list containing 2 elements -- one for the x value, and one for the y value.
list GetEntriesWithYValue(text y)
Returns all entries of the data series matching the specified y value. The returned value is a list, where each element of the list is a list containing 2 elements -- one for the x value, and one for the y value.
ImportFromCloudDB(component clouddb, text tag)
Imports data from the specified CLoudDB component by taking the value identfied by the specified tag value. The expected CloudDB value is a list formatted in the same way as described in the ImportFromList method description.
ImportFromDataFile(component dataFile, text xValueColumn, text yValueColumn)
Imports data from the specified DataFile component by taking the specified x column for the x values, and the specified y column for the y values. The DataFile's source file is expected to be either a CSV or a JSON file.
ImportFromList(list list)
Imports the data from the specified list parameter to the data series. The list is expected to contain element which are also lists. Each list element is expected to have 2 values, the first one being the x value, and the second one being the y value. Invalid list entries are simply skipped.
ImportFromTinyDB(component tinydb, text tag)
Imports data from the specified TinyDB component by taking the value identfied by the specified tag value. The expected TinyDB value is a list formatted in the same way as described in the ImportFromList method description.
ImportFromWeb(component web, text xValueColumn, text yValueColumn)
Imports data from the specified Web component by taking the specified x column for the x values, and the specified y column for the y values. Prior to calling this method, the Web component's Get method has to be called. The expected response of the Web component is a JSON or CSV formatted file.
RemoveDataSource
Call this method to remove the currently attached from the data series. Doing so will result in no more updates from the Data Source being sent to the data series on which the method was called.
RemoveEntry(text x, text y)
Call this method to remove an entry matching the specified x and y values. See AddEntry for an explanation of the valid entry values.
